# Work Permit PDF — Design Reference

This folder is the canonical visual reference for the bilingual (English / Arabic) work permit PDF generated by `supabase/functions/generate-permit-pdf`.

When changing the PDF design, update the HTML mockup first, get visual approval, then port to pdf-lib drawing code.

## Files

- `work-permit-pdf-template.html` — full A4 two-page mockup using sample permit data. Open in a browser to preview.

## Layout overview

**Page 1 — Permit details**

1. **Top header**: Arabic title (تصريح عمل) above English title (Work Permit Form) on the left. Top-right has four bilingual location checkboxes (Business Tower / Shopping Center / Carpark / Outdoor) with the active one filled.
2. **Doc ID strip**: four-cell strip with Permit No., Work Type, Urgency, Issued date. Each cell has English label + Arabic label + value.
3. **SECTION A — PERMIT DETAILS** (black bar):
   - **1. CLIENT / CONTRACTOR DETAILS** (burgundy bar): 3-column grid with requester, contractor, contact fields.
   - **2. WORK DESCRIPTION** (burgundy bar): description text block + start/end/location row.
4. **SECTION B — APPROVAL CHAIN** (black bar):
   - **3. APPROVAL CHAIN** (burgundy bar): per-step rows with role, signer, status badge, signature area.

**Page 2 — Attachments + activity log**

5. **Continuation header**: refers back to the source permit number.
6. **SECTION C — ATTACHMENTS** (black bar):
   - **4. ATTACHED DOCUMENTS** (burgundy bar): 3-column grid of attachment cards (thumbnail + type pill + filename + uploader + date + size).
7. **SECTION D — ACTIVITY LOG** (black bar):
   - **5. ACTIVITY TIMELINE** (burgundy bar): chronological list of actions with date+time, colored marker, and actor.

## Typography

- **English body**: Hanken Grotesk (400/500/600/700/800)
- **Arabic body**: Noto Sans Arabic (matches reference Hot Works form style)
- **Numerics**: monospace system stack (SF Mono / Menlo) for permit numbers, civil IDs, dates

For the actual PDF, the pdf-lib function embeds these via TTF — pdf-lib doesn't fetch Google Fonts at runtime.

## Page structure

- A4: 595 × 842 PDF points (matches 794 × 1123 CSS pixels at 96dpi)
- Page margin: 22pt on all sides
- Subtle triangular pattern in the gutter area outside the content frame

## Bilingual pattern

Every label appears in both English (uppercase, tracked) and Arabic (Noto Sans Arabic, RTL). Values themselves (names, dates, IDs) are universal — no duplicate value rendering.

Section bars carry the English title left-aligned and the Arabic equivalent right-aligned in the same bar.

## Visual reference

This is the design the `generate-permit-pdf` edge function aims to produce. When the function's output diverges from this reference, the function should be updated to match — not the reference.
